### 文件职责

- `main.py`
  - 程序入口。
  - 初始化配置、硬件驱动、service、Web 服务。
- `config.py`
  - 统一管理 GPIO 引脚、数据库路径、摄像头参数、调试开关等配置。

### drivers

- `drivers/pn532.py`
  - `PN532Reader` 类。
  - 提供 UID 读取、轮询过滤和基础设备信息能力。
- `drivers/servo.py`
  - `Servo` 类。
  - 提供通用舵机控制，可复用于门锁舵机和摄像头支架舵机。
- `drivers/button.py`
  - `Button` 类。
  - 提供按钮状态读取和按下检测。
- `drivers/ultrasonic_sensor.py`
  - `UltrasonicSensor` 类。
  - 提供柜内距离测量能力。
- `drivers/buzzer.py`
  - `Buzzer` 类。
  - 提供开关、短鸣、指定时长鸣叫。
- `drivers/rgb_led.py`
  - `RgbLed` 类。
  - 提供设定颜色、关闭、闪烁。
- `drivers/camera.py`
  - `CsiCamera` 类。
  - 提供拍照、取帧、视频流配置与摄像头参数控制能力。

### services

- `services/access_service.py`
  - 卡权限管理与开门鉴权。
  - 支持授予 / 取消某张卡开门权限。
- `services/locker_service.py`
  - 柜门相关业务编排。
  - 协调 RFID、门锁舵机、日志、状态切换。
- `services/camera_service.py`
  - 视频流、摄像头参数、抓拍的业务编排。
- `services/vision_service.py`
  - 人脸检测、识别框结果、自动抓拍。
- `services/camera_mount_service.py`
  - 摄像头上下 / 左右两个舵机的待机、追踪、归位和扫描逻辑。
- `services/occupancy_service.py`
  - 基于超声波测距做柜内空 / 满判定。
- `services/email_service.py`
  - 按钮触发邮件通知与重复发送过滤。

### web

- `web/routes_control.py`
  - 对前端暴露开门、关门、蜂鸣器、RGB、摄像头转动等控制接口。
- `web/routes_stream.py`
  - 对前端暴露视频流和视觉框数据接口。
- `web/routes_cards.py`
  - 对前端暴露卡录入、卡权限管理接口。
- `web/routes_logs.py`
  - 对前端暴露日志和抓拍记录查询接口。
- `web/schemas.py`
  - Web API 的请求 / 响应数据结构，包括视频参数和视觉框数据。

### data

- `data/event_store.py`
  - SQLite 初始化、业务表写入、查询聚合。
- `data/schema.sql`
  - 当前数据库表结构定义。

### scripts

- `scripts/hardware_smoke_test.py`
  - 单独测试每个硬件驱动是否可用。

### 当前推荐开发顺序

- [ ] 先补部署与运维文档，保证树莓派重启后可以自恢复启动。
- [ ] 再做整机稳定性验证，优先看 `PN532 + camera + pigpio` 同时运行时的表现。
- [ ] 然后补剩余实机调参项，重点是超声波阈值和 LED / 蜂鸣器体验。

### 驱动层约束

- [ ] `drivers/` 只负责直接操作硬件，不处理业务规则。
- [ ] 每个驱动尽量封装成独立类，接口保持简单直接。
- [ ] 后续业务逻辑统一放到 `services/`，避免 GPIO 细节散落到路由或主流程里。

## Cross-Phase Rules

- [ ] 每完成一个 phase，都更新 README / TODO，避免文档落后于代码。

- [ ] 先打通最小闭环，再扩功能，不同时并进太多未验证的模块。
- [ ] 不在驱动层直接混入业务逻辑、数据库、HTTP。
